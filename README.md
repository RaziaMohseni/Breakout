# Breakout Game 🎮

یک نسخه‌ی **کنسولی** از بازی کلاسیک Breakout (آجرشکن) به زبان ++C، توسعه‌یافته به
عنوان پروژه‌ی دانشگاهی. بازی روی ترمینال ویندوز اجرا می‌شود و از کاراکترهای متنی
برای رندر و از Console API ویندوز برای ورودی و صدا استفاده می‌کند.

## 🚀 ویژگی‌ها

- منوی اصلی با ۶ گزینه: New Game، Load Game، Help، History، Settings، Exit
- سیستم رکورد (High Score) با ذخیره‌سازی دائمی در `highscore.txt`
- سه سطح سختی قابل تنظیم از طریق منوی Settings
- نمایش زنده‌ی امتیاز، رکورد و تعداد جان‌ها (HUD)
- جلوه‌ی صوتی هنگام از دست رفتن جان (`Beep`)
- معماری ماژولار: جداسازی منطق بازی، رندر و ساختارهای داده

## 🎯 مشخصات بازی

| پارامتر        | مقدار            |
|----------------|------------------|
| عرض زمین       | `BOARD_WIDTH`    |
| ارتفاع زمین    | `BOARD_HEIGHT`   |
| تعداد جان‌ها    | `INITIAL_LIVES`  |
| موقعیت اولیه پدال | مرکز پایین زمین |

## 🕹️ کنترل‌ها

### داخل بازی
| کلید   | عملکرد                  |
|--------|-------------------------|
| `A`    | حرکت پدال به چپ          |
| `D`    | حرکت پدال به راست        |
| `Esc`  | بازگشت به منوی اصلی      |

### منو و تنظیمات
| کلید      | عملکرد              |
|-----------|---------------------|
| `W`       | حرکت به بالا        |
| `S`       | حرکت به پایین       |
| `Enter`   | انتخاب / تأیید      |

## ⚙️ سطوح سختی

سطح سختی از طریق متغیر `GAME_SPEED` کنترل می‌شود (عدد کمتر = سریع‌تر):

| سطح    | سرعت |
|--------|------|
| Easy   | `50` |
| Normal | `35` |
| Hard   | `20` |

## 🧠 منطق بازی

حلقه‌ی اصلی (`runGame`) در هر تکرار:
1. وضعیت کلیدها را با `GetAsyncKeyState` می‌خواند و پدال را حرکت می‌دهد
2. توپ را با `updateBall` به‌روزرسانی می‌کند
3. برخوردها را با `checkCollisions` بررسی می‌کند
4. در صورت بیشتر شدن امتیاز از رکورد، آن را به‌روز می‌کند
5. شرط برد را با `checkWin` بررسی می‌کند
6. اگر توپ از زمین خارج شود: یک جان کم، صدای `Beep`، و سرویس مجدد توپ
7. رندر صحنه و HUD، سپس `Sleep(GAME_SPEED)`

در پایان بازی، امتیاز نهایی با `saveHighScore` در فایل ذخیره می‌شود.
